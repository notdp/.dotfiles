# Correlated Decisions: 把"就这一次"看成一整类未来选择

本文记录一条零散参考：用"相关决策"（correlated decisions）的视角重新解释坏习惯为什么难改，并把它转成自我管理与 AI 编程纪律的工具。

Reference:
- [未验证] 用户摘录，2026-05-29。原文标题和 URL 未提供，定位为自我管理技巧而非严格决策理论。

## 1. 核心观点

你现在这一次选择，不只是一次选择。因为未来的你大概率会在类似场景下用同一套决策算法，所以这一次其实代表了一整类未来选择。

想刷手机 / 抽烟 / 熬夜 / 吃垃圾食品时，常见借口是"就这一次"。从相关决策的角度看，这不成立：未来的你和现在的你高度相关，如果现在的决策算法输出"可以破例"，未来类似时刻很可能也输出"可以破例"。所以应该把眼前这一次看成很多次选择压缩在一起。

心理效果：它让"今天忍一下"不再只是牺牲眼前快乐，而变成"我在训练未来很多个自己也更容易做出好选择"。

## 2. 不想完全戒掉、只想减少频率时

- **随机化**：想做坏习惯时掷骰子，比如只允许 1/3 的冲动被执行。
- **排期**：提前规定可以做的时间（如一周一次），而不是每次靠临场意志力。

评论区补充：

- **坏习惯预算**：比如一年只买固定数量的烟，抽完就没了。
- **自愿税**：每次坏习惯加一笔成本，比如每抽一支烟就转一笔钱出去。

前提：这些方法要求坏习惯事件边界清楚；像"饮食不健康"这种模糊习惯较难执行。

## 3. 对工作 / AI 编程的应用

不要问"这一次做了会怎样"，而要问"如果我在这种状态下总是这么决策，会怎样？"

| "就这一次"的叙事 | 实际在训练的默认算法 |
|---|---|
| "这次不写测试没事" | 以后也跳过验证 |
| "这次先让 AI 糊一下" | 持续增加认知债务 |
| "这次先不读 diff" | 建立不理解代码也合并的习惯 |
| "这次先开个新坑" | 强化逃离主线的模式 |

## 4. 判断

这篇不是严格的决策理论，更像一个自我管理技巧。最有用的地方是把"破例一次"的叙事改成"我正在决定类似场景下的默认算法"。

它可以和"今天决定不做什么"一类方法连起来：前者是减少选项，本文是提高每次微小选择的权重。

一句话总结：坏习惯之所以难改，是因为"就这一次"其实会和未来无数次类似选择相关；把眼前选择看成在训练未来自己的默认决策算法，会让好选择在心理上更有分量。

## 6. Adoption level

当前采用级别：**L1 Document（observe）**。

- L1: 作为决策纪律的背景参考保留。
- 不向 L4/L5 提升：这是软性自我管理框架，写进 `agents/AGENTS.md` 全局规则或 hook 提示会违反本仓库 anti-bloat 纪律（"不要把解释性内容堆进 AGENTS.md""能靠脚本/hook 强制的不要只写自然语言"），也违反下面的 premise collapse。最多以 observe 注记映射到 `guard-close`，由人判断是否引用。

## 7. Premise collapse

如果把本文理解成"任何一次破例都等于永久堕落"或"必须用规则强制每个微选择"，就过度外推了。更窄的结论是：把单次选择看成"在训练未来的默认算法"是一种**叙事/权重调整工具**，用来提高好选择的心理分量；它不替代具体场景的成本收益判断，也不应升级为强制门禁。
