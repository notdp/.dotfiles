# Learning loop

## 一条 learning note 至少回答 5 个问题

1. 这次处理的对象是什么？
2. 为什么之前会卡住或容易踩坑？
3. 最终采用了什么做法？
4. 哪部分可以复用到下次？
5. 还有哪些边界或后续项没解决？

## 写作约定

- 标题写问题域，不写空泛感想
- 关键术语保留英文原词，必要时补中文解释
- 面向团队共享时，摘要可以双语，但正文先求清晰再求花哨
- 经验必须附证据：测试、命令输出、代码路径、截图之一

## 优先沉淀的内容

- checklist
- decision rule
- reusable snippet
- anti-pattern
- verification recipe
