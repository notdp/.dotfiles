# Ideate

定位:**唯一以"发散、扩大解空间"为目标函数的 think 技能。** 其余 think-* 全是收敛型(理解 / 对齐 / 调研下结论 / 取舍 / 成文 / 救火)。本 skill 只负责把候选撑多、撑杂、反惯性,再移交收敛技能裁决。**只产候选,不下结论、不打分。**

真实痛点:面对开放/创造性问题,模型(和人)会过早锁定第一个合理方案,解空间没撑开就收敛。本 skill 用流程纪律强制"先广后窄",把"想到更多更好的方向"变成可执行步骤而非靠灵感。

## 触发与不触发(防误路由)

| 场景 | 用本 skill? | 否则用谁 |
|---|---|---|
| 健康态,主动想要更多、更不一样的方案/点子 | ✅ | — |
| 已有明确候选,要按维度打分取舍 | ❌ | `think-compare` |
| 要在已知方案里选型并下推荐结论 | ❌ | `think-research` |
| 把已定方向落成 spec / 计划 | ❌ | `think-plan` |
| 连续失败 2 次 / 卡壳 / 路径漂移 | ❌ | `think-unstuck` |
| 开放主题的多源资料综述(不产新方案) | ❌ | `think-survey` |
| 先把"要解决什么问题"锚定清楚 | ❌ | `think-scope` |

本 skill 与 think-plan §2、think-unstuck 的"类比 / 第一性原理 / 简化"有技法重叠,**区别在朝向**:那两个是为收敛到单一方案服务的解题框架;本 skill 把这些技法**反过来用于撑大解空间**,且加一套反过早收敛纪律。

## 0. 锚定发散目标

- 一句话写清要为什么发散:要解决的问题 / 要达成的目标 / 不可牺牲约束。
- 定 N:本轮至少产出几条**本质不同**的候选(默认 N≥5)。
- 声明发散边界:哪些约束是真硬约束(不可碰),哪些是可被挑战的"假约束"(发散时优先质疑)。

## 1. 发散纪律(反过早收敛,硬规)

- **生成与评判时间分离**(de Bono 绿帽 / 黑帽):本阶段只生成,**禁止任何"这个不行/不现实"的批判**。批判留到收敛技能。
- **禁自我审查**:不因"听起来蠢/不可行"就不写出来;反直觉候选至少留 1 条。
- **强制 N**:不到 N 条本质不同候选不结束;"换个措辞"不算一条新候选。
- **先发散问题,再发散方案**(双菱形):先问"这问题还能怎么重新定义",再对每种定义发散方案——避免在错误问题上堆点子。

## 2. 发散技法(按问题选用,不必全跑)

| 技法 | 怎么用 | 治什么 |
|---|---|---|
| SCAMPER 七镜 | 对现有方案做定向变异:替换/合并/适配/改放大/他用/删减/反转重排 | 在已有对象上系统穷举变体 |
| 形态分析(Zwicky 箱) | 把问题分解成几个**独立维度**,各列取值,跨维度组合 | **治"换皮创意"**:逼出没想到的组合,而非自发联想跳点 |
| 类比 / 仿生迁移 | "别的领域/自然界怎么解决同构问题" | 跳出本领域惯性 |
| 约束反转 / 强加 / 极端化 | 反转一个假约束;强加一个荒谬约束;把某参数推到极端 | 打破固化前提 |
| 随机刺激 / PO 挑衅 | 引入随机词或故意荒谬陈述,不判真假只看引向何处 | 打断自组织的固化路径 |
| 反向头脑风暴 | "怎么让它彻底失败/搞砸"→ 反推规避项 | 从失败面挖出正向候选 |

## 3. 多样性自检(治 LLM 伪创意,硬规)

发散结束前自检,不通过就再发散一轮:

- **同源检测**:这些候选是否都来自同一个底层思路?只是换皮?——是则砍掉重复项,补真正不同维度的。
- **维度覆盖**:候选覆盖了几个不同的方向/视角/技法?单一来源即不合格。
- **反惯性**:有没有至少 1 条"第一反应不会想到"的候选?没有则用第 2 节技法再撑。

## 4. 轻收敛(只到"可移交",不裁决)

- 把候选聚成 2-3 个有区分度的方向,**每个标差异点和适用前提**,但**不打分、不给推荐**。
- 谁更好的判断**显式留给** think-compare / think-research——这是边界,不能越。

## 输出格式

```markdown
## 发散目标
[问题/目标 + 不可牺牲约束 + 被挑战的假约束 + N]

## 候选清单(本质不同,不评判)
| # | 候选 | 来自技法/视角 | 差异点 | 适用前提 |
|---|------|------------|--------|---------|

## 多样性自检
- 同源检测:<是否有换皮重复;砍了哪些>
- 维度覆盖:<覆盖了几个不同方向>
- 反惯性:<反直觉候选>

## 轻收敛(不裁决)
- 方向 A:<聚类 + 差异 + 适用前提>
- 方向 B:...

## 移交
- 要打分取舍 → /think-compare
- 要选型下结论 → /think-research
- 要落 spec → /think-plan
- 发散中发现其实是卡壳 → /think-unstuck
```

材料/约束不足时,先列缺口再给低置信度候选并标 `[推断]`,不臆造。

## Gotchas

- 别在本 skill 里给最终推荐或打分——越界 think-compare,且会让"发散"提前收敛。
- 别用换措辞冒充新候选;形态分析 + 多样性自检就是来拦这个的。
- 别边发散边批判;批判留给收敛技能,否则好点子一冒头就被掐死。
- 反直觉候选不是凑数;它常是真正撑开解空间的那条。

## 禁止

- 给最终方案推荐 / 维度打分 / 选型结论(越界,移交 think-compare / think-research)。
- 直接进入 spec 或实现(移交 think-plan)。
- 跳过多样性自检(候选同源也要显式声明已自检)。

## 关联技能

- 发散后取舍 → `/think-compare`
- 发散后选型下结论 → `/think-research`
- 方向定了落计划 → `/think-plan`
- 发散中发现是卡壳/失败升级 → `/think-unstuck`
- 发散前需先锚定真问题 → `/think-scope`
