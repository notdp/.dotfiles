# Observe

定位：写实现时把"出问题能被发现、被定位"做进代码，而不是上线出事后才补。配合 `agents/dev-guideLines.md` 的可观测性规则使用：规则定标准，本 skill 给落地方法和分级判断。语言无关，标准落在行为层（错误是否可发现、关键路径是否可还原），不绑定具体 logging 库。

## 1. 何时该加（触发判断）

不是每段代码都要埋点。命中任一条，这段代码需要可观测性设计：

| 信号 | 为什么 |
|---|---|
| 错误路径 / catch / 失败分支 | 出错时要能定位，否则只剩一个无上下文的 500 |
| 外部调用（DB/HTTP/MQ/RPC/文件/子进程） | 边界最易失败，且失败原因在本进程外 |
| 状态变更 / 写操作 / 副作用 | 数据问题要能追到是哪次写、写了什么 |
| 关键分支 / 重试 / 降级 / 限流 | 线上行为要能还原走了哪条路 |
| 跨服务 / 跨异步边界 | 需要 trace 串联，否则日志各自孤立 |

不命中（纯内部纯函数、无副作用的数据变换、已被上层覆盖的薄包装）默认从简，避免噪音。

## 2. log / trace / metric 选哪个

| 手段 | 回答的问题 | 何时用 |
|---|---|---|
| **Log** | "这一次具体发生了什么" | 错误、关键状态变更、外部调用的入参摘要与结果；单条可读 |
| **Trace** | "一次请求跨组件怎么流动的" | 跨服务/跨异步边界；用 correlation/trace id 串联，关键 span 标注 |
| **Metric** | "整体趋势/速率/分布如何" | 计数、延迟、错误率、队列深度；用于告警和容量，不用于定位单次 |

三者不互斥：一次外部调用失败，通常 log 记这次详情 + metric 计错误率 + trace 标这个 span 失败。

## 3. 错误可发现性（高风险，正反对照）

错误路径是可观测性的第一优先级。静默吞错是最常见的不可观测来源。

| Wrong | 为什么错 | Right |
|---|---|---|
| `catch (e) {}` / `except: pass` | 错误消失，线上无任何痕迹 | catch 后处理，或带上下文记录/重抛：`log.error("付款回调校验失败 order=%s reason=%s", order_id, e)` |
| `catch (e) { return null }` | 调用方拿到 null 不知为何 | 记录原因再返回 fallback，或让错误带类型上抛 |
| `log.error("failed")` | 无定位信息，等于没记 | 带关键变量（id、状态、外部响应码），不含 secret |
| 错误信息含 token/密码/PII | 日志成为泄露面 | 脱敏后记录，只留定位所需字段 |

仅在确实可以忽略错误时（已知可重试的瞬时错误、清理路径的 best-effort）才允许吞，且必须在同处注释说明为什么可以忽略。

## 4. 分级：避免过度埋点

可观测性也有成本（噪音、性能、日志费用）。按影响面给强度，不是一刀切：

- **高（必须）**：错误路径、外部调用边界、写操作/状态变更、跨服务边界
- **中（推荐）**：关键业务分支、重试/降级决策、慢路径入口
- **低（从简或不加）**：纯内部纯函数、无副作用变换、高频热路径里的非关键细节（加了反而拖慢并刷屏）

判据：能说出"这条观测点在排障时会被谁查、回答什么问题"才加；说不出就不加。日志泛滥和无日志一样不可观测。

## 5. 与 dev-operational-task 分工

| 任务 | 用谁 |
|---|---|
| 代码级：错误可发现、关键路径 trace、外部调用日志 | 本 skill |
| 长任务级：批处理进度、ETA、heartbeat、断点续跑、dry-run 数据准确性 | `/dev-operational-task` |

两者可叠加：一个数据回填脚本，进度可观测走 `/dev-operational-task`，单条记录处理失败的错误可发现走本 skill。

## Gotchas

- 沿用项目已有的 logging 设施和格式，不新引入平行 logger 或日志库
- 可观测性代码也是代码：受 Surgical Changes 约束，不借埋点之名顺手改无关逻辑
- 不把可观测性等同于"到处 print"；print 无级别、无结构、难关闭，优先用项目的 logger
- 加 trace id / correlation id 时复用框架既有的上下文传递，不手搓全局变量

## 禁止

- 静默吞错（空 catch / `except: pass` / 吞掉后返回默认值且无记录），除非同处注释说明可忽略的理由
- 在日志/trace/metric label 里写入 secret、token、密码、完整 PII
- 用高频热路径的逐行 log 制造噪音，掩盖真正的关键事件
- 为不可观测的改动留一句"日志后面再加"：错误路径的可观测性是当次产出，不是 backlog
